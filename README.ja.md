GlyphStudioは、決定的なピクセル編集、フレームごとのアニメーション、そして将来的な動作アシスタンスを実現する、クリエイター向けの画像編集ツールです。

GlyphStudioは、**Tauri v2**、**React**、および**Rust**を使用して開発されたデスクトップアプリケーションです。このアプリケーションは、以下の原則に基づいて設計されています。編集機能が常にアートワークを制御し、自動化はアーティストの創造性をサポートする役割に留まるべきである、ということです。

## 現在の状況

GlyphStudioは、30のステージが実装され、1,149のテストが正常に完了している、動作可能なデスクトップエディタです。

### キャンバスエディタ (Rustバックエンド)
- 最近傍法によるレンダリングを行う、決定的なピクセルキャンバス
- レイヤーの表示/非表示、ロック、不透明度、名前変更、並べ替え
- 線の描画機能（アンドゥ/リドゥ対応）
- 長方形の選択、クリップボード操作、変換機能
- 複数フレームに対応したタイムライン（フレームごとのアンドゥ/リドゥ対応）
- 前後のフレームを重ねて表示する機能（オニオンスキン）
- フレームレート（FPS）とループ再生に対応した再生コントロール
- 決定的な提案に基づく動作アシスタンス
- 階層構造、フェードアウト、および二次的なモーションテンプレートを持つアンカーシステム
- 複数のフレームにまとめて適用できるモーションプリセット
- ピボット、タグ、および検証機能を持つクリップ定義
- マニフェストファイル付きのスプライトシートエクスポート（ネイティブおよび汎用ランタイム形式に対応）
- サムネイル表示、検索、およびバンドルパッケージング機能を持つアセットカタログ
- プロジェクトの保存/読み込み、自動保存からの復元、およびスキーマの移行機能

### シーンコンポジタ (フロントエンド + Rust)
- アセットインスタンス、Zオーダー、表示/非表示、不透明度、パララックスによるシーンの構成
- パン、ズーム、キーフレームアニメーション、およびショットの派生機能を持つカメラシステム
- スロット、プリセット、検証、およびシーンブリッジ機能を持つキャラクタービルドシステム
- フルスナップショット履歴を持つシーンのアンドゥ/リドゥ機能、および同期エラー時のロールバック機能
- 20種類の操作に関する詳細な情報を含む、永続的なプロビナンス
- シーンの比較とプレビュー機能

### スプライトエディタ (フロントエンドのみ)
- ペン、消しゴム、塗りつぶし、スポイトなどのツールを備えた、スタンドアロンのピクセルエディタ
- レイヤーごとの表示/非表示、名前変更、並べ替え機能を持つ、複数レイヤーの編集
- すべてのレイヤーでリアルタイムのプレビュー表示を行う、アルファ合成機能
- オニオンスキン、再生、スクラバー、およびフレームごとの持続時間機能を持つ、フレーム管理機能
- コピー/カット/ペースト/削除機能を持つ、長方形の選択
- 複数レイヤーをまとめてエクスポート/インポートできる、スプライトシートのインポート/エクスポート機能
- カラーピッカーと、前景/背景色の入れ替え機能を持つ、パレットパネル

これは、ブラウザ上で動作する簡易ツールや、ランダムな結果を生成するツールではありません。Rustがキャンバスのピクセルデータを管理し、フロントエンドがスプライトのピクセルデータを管理する、ネイティブなデスクトップエディタです。

## 製品の理念

GlyphStudioは、以下の4つの原則に基づいて開発されています。

1. **決定的な編集**
すべてのピクセルの変更は、ルールに基づいて行われ、検査可能で、元に戻すことができます。

2. **サポート的なAI**
自動化は、ワークフローを支援しますが、創造的な自由を制限するものではありません。

3. **アニメーションを重視した構造**
フレーム、タイムライン操作、オニオンスキン、および再生機能は、エディタの中核となる概念であり、後付けの機能ではありません。

4. **信頼性の高い状態管理**
保存/読み込み、自動保存、復元、アンドゥ/リドゥ、および移行は、単なるメンテナンス作業ではなく、製品の重要な機能として扱われます。

## アーキテクチャ

### フロントエンド (React + TypeScript)
- ドメインごとに整理された17以上のZustandストア
- 両方のエディタで使用されるHTMLキャンバスレンダラー
- キャンバスエディタUI: レイヤー、タイムライン、選択、再生、キャラクタービルダー、シーンコンポジタ
- スプライトエディタ: フロントエンドが管理するピクセルバッファを備えた、スタンドアロンのピクセル編集機能
- 複数レイヤーのスプライト編集のための、`flattenLayers`によるアルファ合成

### バックエンド (Rust)
- キャンバスエディタ向けの、信頼性の高いピクセルバッファとレイヤー合成機能
- 変更前後の状態を記録するストローク処理
- 選択/変換機能
- プロジェクトの永続化、自動保存、クラッシュからの復旧機能
- エクスポート機能 (PNG、スプライトシート、クリップ、バンドル)
- カメラと再生機能を備えたシーン構成エンジン
- サムネイル生成機能付きのアセットカタログ
- 実装済みのTauriコマンド：166個

### デスクトップシェル
- Tauri v2

## 実装段階

### 段階1～3：エディタの基礎
キャンバス、レイヤー、描画ツール、アンドゥ/リドゥ、選択、変換、タイムライン、オンionsキン、再生、フレーム操作、PNG/ストリップエクスポート、プロジェクトの永続化、自動保存、クラッシュからの復旧。

### 段階4A：モーションアシスタンス
制約付きモーションセッション、決定的な提案生成、ミニフレームストリップによるプレビュー、セッションの安全性、タイムラインへの提案の適用。

### 段階5～8：モーションの調整
階層構造とフェードアウトを持つアンカー、二次モーションテンプレート（風、揺れ、スイング、ざわめき）、分析メトリックを持つモーションサンドボックス、バッチ適用可能なモーションプリセット。

### 段階9～10：クリップ、エクスポート、シーンの基礎
ピボット/タグ/検証機能付きのクリップ定義、マニフェスト付きのスプライトシートエクスポート、サムネイル付きのアセットカタログ、バンドルパッケージング、インスタンスとZオーダーを備えたシーン構成。

### 段階11～14：キャラクターシステム
12個のボディ領域スロットを持つキャラクタービルド、互換性レベルを持つプリセットピッカー、ビルドの検証、永続化されたビルドライブラリ、シーンへのキャラクターのブリッジ機能（スナップショット配置）。

### 段階15～16：シーンの編集
パン/ズーム機能付きのシーンカメラ、補間付きのカメラキーフレーム、フルスナップショット履歴を持つシーンのアンドゥ/リドゥ、バックエンド同期エラー時のロールバック。

### 段階17～24：プロビナンスと検査
20種類の操作を含むシーンのプロビナンスの永続化、キャプチャされた変更前後のスライスによる詳細検査、構造化された値の概要、シーン比較エンジン、プレビューワークフローの復元。

### 段階25～26：復元と選択的復元
完全な派生に基づくシーンの復元契約、インスタンス、カメラ、キーフレーム、再生など、ドメインごとの選択的復元、アンドゥ/リドゥ機能付きのlawful seamによるplaybackConfigの設定。

### 段階27～28：スプライトエディタ
フロントエンドのみのスプライトエディタ：ドキュメント契約、鉛筆/消しゴム/塗りつぶし/スポイト機能付きのピクセルキャンバス、オンionsキン付きのフレーム、クリップボードによる選択、スプライトシートのインポート/エクスポート、キーボードショートカット、ズーム/グリッド、パレットパネル。

### 段階29：アニメーションプレビュー
アニメーションプレイヤー契約、スクラバーとスペースショートカット付きの再生UI、プリセット付きのインラインフレーム期間編集、再生中のオンionsキンの抑制。

### 段階30：レイヤーとレイヤーワークフロー
SpriteLayerタイプ、レイヤーIDでキー付けされたピクセルバッファ、flattenLayersによるアルファ合成、activeLayerIdの追跡、CRUD/可視性/名前変更/並べ替え機能付きのレイヤーパネル、すべての可視レイヤーにまたがるドラフトストローク合成、マルチレイヤーエクスポート。

## エクスポートサポート

### キャンバスエディタ (Rust)
- **Current Frame PNG** — 結合された単一画像
- **PNG Sequence** — 連番のファイル (name_0001.png, name_0002.png, ...)
- **Sprite Strip** — 水平または垂直方向に並んだ単一画像
- **Clip Sheet** — クリップ定義から作成されたスプライトシート。オプションでマニフェストファイルも含まれます。
- **All Clips Sheet** — 有効なすべてのクリップを結合したシート
- **Asset Bundle** — 画像、マニフェストファイル、およびプレビューサムネイルを含むフォルダまたはzipファイル
- **Catalog Bundle** — 各アセットごとにサブフォルダが作成された、複数のアセットをまとめたパッケージ

### Sprite Editor (フロントエンド)
- **Sprite Strip** — フレームごとにすべての表示レイヤーを結合した水平方向のストリップ
- **Current Frame** — 表示レイヤーを結合した画像

エクスポートは、結合された表示レイヤーのみを使用します。 オニオンスキン、再生状態、および一時的なエディタのオーバーレイは、出力には含まれません。

## 目標外の機能

GlyphStudioは、以下の機能を提供することを目的としていません。

- 一般的な画像編集ソフト
- ブラウザ上で動作する簡易アプリケーション
- AIによるプロンプト解釈でアートを生成するツール
- フロントエンドの状態とバックエンドの真実が乖離する不安定な環境

## セキュリティ

GlyphStudioは、**デスクトップ環境でのみ動作する**アプリケーションです。 ネットワークリクエストを行わず、テレメトリデータを収集せず、機密情報を扱いません。

- **アクセスするデータ:** ローカルのスプライトファイル (.glyph, .pxs, .png)、アプリケーションデータディレクトリ内の自動保存/復元ファイル
- **アクセスしないデータ:** ネットワーク接続なし、クラウドサービスとの連携なし、リモートAPIの利用なし、ユーザーアカウントの管理なし
- **権限:** Tauri v2のネイティブファイルダイアログを通じて、ユーザーが選択したディレクトリへのファイルシステムアクセスのみ許可
- **テレメトリデータは収集および送信されません。**

脆弱性に関する報告は、[SECURITY.md](SECURITY.md) を参照してください。
