# Why GlyphStudio

GlyphStudio is a desktop pixel asset studio. Not just an editor — a production system for stills, motion, variants, and structured output.

## What makes it different

**Variant workflows, not just recoloring.** Create named palette sets (Fire, Ice, Seasonal) and document variants (walk-left, walk-right, idle). Preview before commit. Export the whole family in one pass with contextual filenames.

**Reusable parts, not clipboard tricks.** Promote any selection to a named, thumbnailed, searchable part. Stamp it across frames and projects. Save curated parts as packs for cross-project reuse.

**Structured output, not manual repetition.** Bundle export combines base + document variants + palette variants into a single deliberate output pass. Filenames are predictable: `{name}-{variant}-{palette}.{ext}`.

**Templates and packs, not blank starts.** Start from an animation-ready template, a variant-ready setup, or a curated asset pack. ProjectHome reflects real production workflows, not just canvas dimensions.

**Animation as a first-class system.** Frame-by-frame editing with onion skin, per-frame timing holds, playback, and seam-quality tools. Not an afterthought bolted onto a still-image editor.

**Honest tools.** Parts are committed pixels (stamped), not fake linked instances. Variants are independent forks, not hidden shared state. Preview is transient — it never silently mutates your work.

## In one sentence

GlyphStudio lets you author a base pixel asset, create variants, build reusable parts, and export the resulting family — all in one governed workflow.

## Built with

- **Tauri v2** — native desktop, cross-platform
- **React** — UI and state management
- **Rust** — pixel truth, persistence, export pipelines
- **MCP server** — 76 programmable tools for LLM-assisted workflows
